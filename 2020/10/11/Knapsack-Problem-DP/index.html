Knapsack Problem (DP)

Knapsack Problem (DP)
by ZhangZiheng

In this blog, I’ll review the classic problems of dynamic programming: Knapsack Problem.

0-1 knapsack problem

Description

There are $N$ items(only one number of each) and a knapsack of capacity $V$. Each item $i$ has weight $C_{i}$. How to get the maximum total value in this knapsack? Note: You can’t break an item.

$0<N,V≤1000$.

Solution

This is a basic Knapsack Problem. In this problem, for each item, you can choose to pick the complete item(1) or don’t pick it(0), so we call it the 0-1 knapsack problem.

Let $f(i,j)$ be a function which represents the maximum value that first $i$ items can be put into a knapsack with a capacity of $j$. So we can get the following equation.

In equation ($\ref{equ:1}$), $f(i-1,j)$ means we don’t pick the $i^{th}$ item, and in this situation, it’s equivalent to putting the first $i-1$ items into a knapsack with capacity $j$.

The other expression $f(i-1,j-C_i)+W_i$ means we pick the $i^{th}$ item. So the remaining capacity of knapsack is $j-C_i$, and the total value is $f(i-1,j-C_i)+W_i$.

Implementation

  • Time complexity $O(NV)$, space complexity $O(NV)$.
#include <bits/stdc++.h>

using namespace std;

const int N = 1e3 + 10;

int f[N][N];
int v[N], w[N];
int n, m;

int main()
{
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> v[i] >> w[i];
    
    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= m; j++)
        {
            f[i][j] = f[i - 1][j];
            if (j >= v[i]) f[i][j] = max(f[i][j], f[i - 1][j - v[i]] + w[i]);
        }
    }
    cout << f[n][m] << endl;
    return 0;
}

  • Obviously, $f(i,\dots)$ is only related to $f(i-1,\dots)$, so we can optimize space complexity to reduce it from $O(NV)$ to $O(V)$.

    Maybe it’s confused about why we need to enumerate knapsack weights in reverse order.

    We can look at the following optimized equation ($\ref{equ:2}$).

    In the 2D equation ($\ref{equ:1}$), $f(i,j)$ depends on the previous result, i.e. $f(i-1, \dots)$.

    In equation ($\ref{equ:2}$), we can find out that $f(j)$ is related to $f(j-C_i)$. If we enumerate weights from small to large, when we need to get $f(j)$, because of $j-C_i < j$, $f(j-C_i)$ has already been overwritten, so $f(j-C_i)$ is not equivalent to $f(i-1, j-C_i)$ in 2D equation.

    When we enumerate weights in reverse order, we can find when we need to calculate $f(j)$, $f(j-C_i)$ is still the result of $(i-1)^{th}$ loop, so in this case, $f(j-C_i)$ is equivalent to $f(i-1, j-C_i)$ in 2D equation.

#include <bits/stdc++.h>

using namespace std;

const int N = 1e3 + 10;

int f[N];
int v[N], w[N];
int n, m;

int main()
{
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> v[i] >> w[i];
    
    for (int i = 1; i <= n; i++)
    {
        for (int j = m; j >= v[i]; j--)
        {
            f[j] = max(f[j], f[j - v[i]] + w[i]);
        }
    }
    cout << f[m] << endl;
    return 0;
}

Complete Knapsack Problem

Description

The only difference between Zero-One Knapsack Problem and Complete Knapsack Problem is the number of items. In the Complete Knapsack Problem, there are infinite number of items.

Solution

The way to think about this problem is the same as the last.

Let $f(i,j)$ be a function which represents the maximum value that first $i$ items can be put into a knapsack with a capacity of $j$.

Equation ($\ref{equ:3}$) means we pick the $i^{th}$ item for $k$ times. So, for the first $(i-1)^{th}$ items, the capacity remains $j-kC_i$. And the total value is $f(i-1,j-kC_i+KW_i)$.

Implementation

#include <bits/stdc++.h>

using namespace std;

const int N = 1e3 + 10;

int n, m;
int f[N][N], v[N], w[N];
int dp[N];

int main()
{
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> v[i] >> w[i];
    for (int i = 1; i <= n; i++)
    {
        for (int j = 0; j <= m; j++)
        {
            for (int k = 0; k * v[i] <= j; k++)
            {
                f[i][j] = max(f[i][j], f[i - 1][j - k * v[i]] + k * w[i]);
            }
        }
    }
    cout << f[n][m] << endl;
    return 0;
}

  • Time complexity is $O(NV\sum\frac{V}{C_i})$. In the worst case, time complexity is $O(NV^2)$ which is a big number.

  • There are another tricks to reduce space complexity from $O(NV)$ to $O(V)$.

    It’s the same optimization as in the Zero-One Knapsack Problem. When we choose the $i^{th}$ item, $f(i,j)$ is only related the $f(i,\dots)$. So in this case, we don’t need to enumerate weights in reverse order. The optimized equation is shown below:

#include <bits/stdc++.h>

using namespace std;

const int N = 1e3 + 10;

int n, m;
int f[N], v[N], w[N];

int main()
{
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> v[i] >> w[i];
    for (int i = 1; i <= n; i++)
    {
        for (int j = 0; j <= m; j++)
        {
            if (j >= v[i]) f[j] = max(f[j], f[j - v[i]] + w[i]);
        }
    }
    cout << f[m] << endl;
    return 0;
}
