[leetcode]10.Regular Expression Matching

Regular Expression Matching
by ZhangZiheng

Description

Given an input string s and a pattern p, implement regular expression matching with support for . and *.

  • . Matches any single character.
  • * Matches zero or more of the preceding element.

The matching should cover the entire input string (not partial).

Note:
s could be empty and contains only lowercase letters a-z.
p could be empty and contains only lowercase letters a-z, and characters like . or *.

// example
Input:
s = "aa"
p = "a"
Output: false
Explanation: "a" does not match the entire string "aa".

Input:
s = "aa"
p = "a*"
Output: true
Explanation: '*' means zero or more of the preceding element, 'a'. Therefore, by repeating 'a' once, it becomes "aa".

Input:
s = "ab"
p = ".*"
Output: true
Explanation: ".*" means "zero or more (*) of any character (.)".

Input:
s = "aab"
p = "c*a*b"
Output: true
Explanation: c can be repeated 0 times, a can be repeated 1 time. Therefore, it matches "aab".

Input:
s = "mississippi"
p = "mis*is*p*."
Output: false

Solution

This problem is dynamic programming, or dp. Why dp is always difficult? Because it depends on our experience. If you are not familiar with some typical dp models, you may feel challenging solve these problems.

In this problem, to make it easier to understand, array index start at 1. (s = ' ' + s, p = ' ' + p;)

In dp problem, we have three steps.

  1. Give dp array a reasonable meaning.

    In our problem, dp array is 2D, i.e. $bool\ dp[i][j]$. This dp array means $s[1\sim i]$ can or can’t match $p[1\sim j]$. We can find inspiration in another classic problem, the longest increasing subsequence. So, that’s why I said dp problem is a little bit difficult. If you don’t know how to solve that problem, you may give a wrong meaning to you dp array.

  2. Find equations to describe each states.

    This may be the most difficult step in dp problems.

    In our problem, we need a classified discussion.

    • $dp[i][j] = dp[i-1][j-1]$ if and only if $s[i]=p[j]$ or $p[j]=\cdot$

    • If $p[j]=\ast$, then

      • If $j\geq 2$, then $dp[i][j]=dp[i][j-2]$, which means we don’t use this $\ast$ and the one character before $\ast$.

      • If $j> 0$ and ($s[i]=p[j-1]$ or $p[j-1]=\cdot$), then $dp[i][j]=dp[i-1][j]$.

        This case is not easy to understand. We can see Fig 1. to help us understand.


        Fig 1. dp(i, j) = dp(i-1, j)

        Let’s think about this question: which state can decide $dp[i][j]$ ?

        In this case, we have already known $s[i]=p[j-1]$ and pattern string $p$ ends in character $\ast$.

        If $p[1\sim j]$ can match $s[1\sim i-1]$, then we can get $dp[i][j]=true$ and vice versa, because character $\ast$ can repeat several times of the preceding element. So as long as $s[i]=p[j-1]$, the ending character of pattern string can repeat until $p$ matches $s$.

Implementation

class Solution {
public:
    bool isMatch(string s, string p) {
        int n = s.size(), m = p.size();
        s = " " + s, p = " " + p;
        vector<vector<bool>> f(n + 1, vector<bool>(m + 1));
        f[0][0] = true;
        for (int i = 0; i <= n; i ++ )
            for (int j = 1; j <= m; j ++ ) {
                if (j + 1 <= m && p[j + 1] == '*') continue;
                if (i && p[j] != '*') {
                    f[i][j] = f[i - 1][j - 1] && (s[i] == p[j] || p[j] == '.');
                } else if (p[j] == '*' && j >= 2) {
                    f[i][j] = f[i][j - 2] || i && f[i - 1][j] && (s[i] == p[j - 1] || p[j - 1] == '.');
                }
            }

        return f[n][m];
    }
};
